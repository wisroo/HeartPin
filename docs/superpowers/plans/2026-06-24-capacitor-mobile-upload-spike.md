# Capacitor Mobile Upload Spike Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a Capacitor dev-app spike that proves whether Android and iOS native photo selection preserves EXIF/GPS metadata for HeartPin uploads.

**Architecture:** Keep the single React/Vite repo and reuse `MobileApp`. Add a small media picker boundary under `src/platform/media/`, route `MobileUploadFlow` through that boundary, then add Capacitor Android/iOS shells for real-device testing. The spike records observed metadata results before Supabase upload work continues.

**Tech Stack:** React 18, Vite 6, Vitest, Capacitor, `@capacitor/camera`, `@capacitor/filesystem`, existing `src/exif.js` JPEG parser, existing `api.uploadPhotos` flow.

## Global Constraints

- Do not split the repository.
- Do not implement Play Store/App Store release.
- Do not implement SupabaseAdapter in this spike.
- Do not permanently upload original photos to Supabase.
- Keep platform differences inside `src/platform/media/` or Capacitor native configuration.
- Web/PWA may still return `lat`/`lng` as `null` on Android browser file input.
- Capacitor dev app must be installable on Android by debug build and on iOS by Xcode dev install.
- `.env.local` uses `VITE_SUPABASE_PUBLISHABLE_KEY`; never place Secret API Key or `service_role` in client env.

```js
export async function readExifFromBytes(bytes, mimeType) {
  if (!bytes || !/jpe?g/i.test(mimeType || "")) return null;
```

```js
import { readExifFromBytes } from "../../exif.js";

export async function normalizeWebFile(file) {
  const buffer = await file.arrayBuffer();
  const bytes = new Uint8Array(buffer);
  const exif = await readExifFromBytes(bytes, file.type);
  return {
    name: file.name,
    mimeType: file.type || "application/octet-stream",
    size: file.size,
    lastModified: file.lastModified,
    bytes,
    takenAt: exif?.date && exif?.time ? `${exif.date}T${exif.time}` : null,
    lat: exif?.lat ?? null,
    lng: exif?.lng ?? null,
    source: "web-input",
    file,
  };
}

export function pickWebPhotos({ source = "library", multiple = true } = {}) {
  return new Promise((resolve, reject) => {
    const input = document.createElement("input");
    input.type = "file";
    input.accept = "image/*,.heic,.heif";
    input.multiple = multiple && source !== "camera";
    if (source === "camera") input.capture = "environment";
    input.onchange = async () => {
      try {
        const files = Array.from(input.files || []);
        const images = files.filter((file) => file.type.startsWith("image/") || /\.(heic|heif)$/i.test(file.name));
        resolve(await Promise.all(images.map(normalizeWebFile)));
      } catch (error) {
        reject(error);
      }
    };
    input.click();
  });
}
```
